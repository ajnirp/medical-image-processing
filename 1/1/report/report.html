Part (d)

If we choose a very small value for delta(t) and/or delta(theta), we would send a lot of samples, which means a lot of radiation. This prolongs the patient's exposure to radiation, which is to be avoided. Also, it increases the time and cost of the CT procedure. However, there is an upper bound to delta(t). When theta is 90 degrees, then a change in delta(t) is most likely to skip a pixel. So therefore delta(t) should not be more than the real-life width of a pixel in the region we are scanning. Otherwise, for theta = 90 degrees (or -90 degrees or 0 degrees or 180 degrees) we might end up skipping an entire line of pixels in the region we are scanning.

Part (e)

Choosing a very large value of delta(s) (relative to pixel width) will result in undersampling. There will be many blank patches in the reconstructed image. Analogously, choosing a very small value of delta(s) (relative to pixel width) will result in oversampling. The resultant image will be extremely bright.
